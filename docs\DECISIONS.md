# Decision log

## ADR-001 — Frontend-only før backend

**Status:** Accepteret  
**Beslutning:** Første version er en statisk salgsdemo.  
**Begrundelse:** Vi skal validere interesse og arbejdsgang før dyr infrastruktur.

## ADR-002 — Super Cut først

**Status:** Accepteret  
**Beslutning:** Alle synlige valg optimeres til Super Cut, selv om komponenterne kan genbruges.  
**Begrundelse:** En skarp nicheoplevelse er mere troværdig end en generisk platform.

## ADR-003 — Lokale fiktive data

**Status:** Accepteret  
**Beslutning:** Ingen rigtige kundedata i repositoryet.  
**Begrundelse:** En offentlig eller delt demo må ikke skabe privatlivsrisiko.

## ADR-004 — GitHub Pages

**Status:** Accepteret  
**Beslutning:** Demoen kan hostes direkte fra repositoryets rod.  
**Begrundelse:** Ingen build, næsten ingen omkostning og enkel deling.

## ADR-005 — Placeholder-billeder

**Status:** Midlertidig  
**Beslutning:** Unsplash bruges indtil Super Cuts billeder er godkendt.  
**Begrundelse:** Designet kan vurderes uden at kopiere Instagram-indhold uden tilladelse.

## ADR-006 — Dansk UI

**Status:** Accepteret  
**Beslutning:** Interface og driftstekst er dansk; hero-tagline må være engelsk.  
**Begrundelse:** Lokal brugbarhed med et redaktionelt brandøjeblik.
