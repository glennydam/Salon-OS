# QA checklist

## Kritiske flows

- [ ] Skift mellem Kundeoplevelse og Salon OS.
- [ ] Åbn booking fra hero, navigation og slut-CTA.
- [ ] Gennemfør alle bookingtrin.
- [ ] Brug tilbageknappen og kontrollér bevaret state.
- [ ] Forsøg at bekræfte uden vilkår og se feedback.
- [ ] Bekræft og luk bookingskærmen.
- [ ] Åbn dashboard, kalender, kunder, opfølgning og muligheder.
- [ ] Søg efter en kunde.
- [ ] Åbn Maria-profilen.
- [ ] Send simuleret genbookingsbesked.

## Layout

- [ ] 390 × 844
- [ ] 768 × 1024
- [ ] 1280 × 800
- [ ] 1440 × 900
- [ ] Ingen utilsigtet vandret scroll.
- [ ] Billeder beskæres men forvrænges ikke.
- [ ] Mobilnavigation dækker ikke vigtigt indhold.

## Tilgængelighed

- [ ] Tastaturnavigation.
- [ ] Synligt fokus.
- [ ] Menuknap opdaterer `aria-expanded`.
- [ ] Formularfelter har labels.
- [ ] Billeder har meningsfuld alt-tekst eller tom alt ved pynt.
- [ ] Reduceret bevægelse respekteres.
- [ ] Farver har læsbar kontrast.

## Teknisk

- [ ] Ingen JavaScript-fejl i konsollen.
- [ ] Alle lokale stier fungerer under GitHub Pages subpath.
- [ ] Ingen secrets eller rigtige kundedata.
- [ ] JSON er gyldig.
- [ ] HTML, CSS og JS indlæses statisk.
- [ ] Eksterne billeder har acceptable fallback-forhold.

## Før offentlig deling

- [ ] Ejer har godkendt brug af navn, adresse og branding.
- [ ] Offentlige fakta er genkontrolleret.
- [ ] Rigtige priser er godkendt eller fortsat mærket demo.
- [ ] Billedrettigheder er dokumenteret.
- [ ] Det er tydeligt, at booking og beskeder er simulerede.
