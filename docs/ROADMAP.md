# Roadmap

## Fase 0 — Konceptdemo

Status: denne repository-version.

- Visuel kundeoplevelse
- Simuleret booking
- Dashboard, kunder og genbooking
- Pilotmanuskript og valideringsspørgsmål

## Fase 1 — Pilotafklaring

- Interview med ejeren
- Bekræft behandlinger, priser, varigheder og åbningstider
- Kortlæg telefonbooking, afbud og genbooking
- Vælg hvilke tre problemer en pilot skal løse
- Aftal billeder, casebrug og datagrænser

## Fase 2 — Minimum driftsklar løsning

Kun ved valideret behov:

- Hjemmeside med godkendt indhold
- Eksisterende bookingsystem eller snæver integration
- Bekræftelser og påmindelser
- Enkle afbudsregler
- Måling af bookinger og afbrydelser

## Fase 3 — Relation og genbooking

- Kundeprofil og behandlingsnoter
- Samtykkestyret opfølgning
- Anmeldelsesflow
- Behandlingsspecifik genbooking

## Fase 4 — Produktbeslutning

Først efter dokumenterede resultater fra flere saloner:

- Multi-tenant arkitektur
- Branchepakker
- Betaling og abonnement
- Support- og driftsmodel
